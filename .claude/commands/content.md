# Content Engine

> **Preamble:** You are generating content for publication. Research before writing — every claim should be data-backed, every angle should be differentiated from the top 5 ranking pages. Match the project's brand voice. Do not publish without human approval.

Generate SEO-optimized content.

**Input:** `$ARGUMENTS` — content type and topic/brief.

---

## Content Types

| Type | When to use |
|------|------------|
| `blog` | Long-form educational content (1500-2500 words) |
| `case-study` | Customer success story with data |
| `landing` | Feature/product landing page copy |
| `comparison` | vs-competitor or vs-alternative |
| `faq` | Frequently asked questions page |

---

## Step 1 — Parse Brief

Extract from $ARGUMENTS:
- Content type
- Topic / target keyword
- Target audience
- Unique angle or data points

If insufficient, ask clarifying questions.

---

## Step 2 — Research

Use WebSearch to gather:
- Top 5 ranking pages for the target keyword
- Related questions (People Also Ask)
- Current trends and data points
- Competitor content gaps

---

## Step 3 — Generate

Write the full content piece including:
- **Title** — SEO-optimized, compelling
- **Meta description** — Under 155 characters
- **Headers** — H2/H3 structure with keyword placement
- **Body** — Value-first, scannable, data-backed
- **CTA** — Clear next step for the reader

---

## Step 4 — HUMAN GATE

Present the draft. Ask:
```
ship  → Save as-is
revise → Specific feedback to incorporate
shorter → Cut length by ~30%
more data → Add more statistics/examples
```

Iterate until approved.

## Rules
- Lead with value, not product features
- Every claim needs supporting data or example
- Write for humans first, search engines second
- Match the existing brand voice if a style guide exists
